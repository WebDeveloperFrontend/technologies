GIT - это система контроля версий. Или другими словами - хранилище, база данных истории разработки проекта.
Такая система нужна для учета всех файлов когда либо созданных в проекте и содержащихся в Git (базе данных).

Команды Git:
git init - Создание нового git репозитория.
git status - Просмотр состояний индекса. Подготовки файлов для комита.
git add <file> - Подготовка файлов для сохранения в репозиторий.
git commit -m "message" - Сохранение изменений. Коммит.
git checkout <commit/branch> - Переход к определенной версии вашего коммита или ветки.
git log - Показывает список всех коммитов.
git log --graph --oneline - Показывает список всех коммитов, в коротком виде.
git branch <branch name> - Создание новой ветки.
git checkout -b <branch name> - Создание новой ветки и переход в нее.
git branch - Отображает список всех веток.
git branch -m <new branch name> - Переименование текущей ветки/
git branch -d <branch name> - Удаление ветки. (текующую ветку удалить нельзя)
git merge <feature branch name> - Слияние веток. main -> m1.

GitHub.
git clone <url> - Клонирование удаленного репозитория в локальный.
git branch -a - Отображает все ветки, включая те, которые находятся в удаленных репозиториях.
git pull - Применение изменений с удаленного репозитория в локальный.
git push - Применение изменений с локального репозитория в удаленный.
git remote add origin <url> - Подключение удаленного репозитория.
git push -u origin <branch> - Загрузка изменений из локальной ветки в удаленную с созданием связи между ними.
